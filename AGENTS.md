# 平面与平面垂直（第一课时）H5交互课件 - 需求拆解文档

## 产品概述

- **产品类型**: 高中数学H5交互课件（立体几何教学工具）
- **场景类型**: <scene_type>prototype-app</scene_type>
- **目标用户**: 高中数学教师（课堂教学使用）+ 高中学生（自主学习/复习）
- **核心价值**: 通过拖拽、滑动、点击等丰富交互，将抽象的面面垂直概念具象化，降低立体几何理解门槛，支持课堂投影与移动端多场景使用
- **界面语言**: 中文
- **主题偏好**: user_specified（蓝色主色调，简洁教育风；支持明暗模式切换）
- **导航模式**: 锚点导航 + 线性浏览双模式
- **导航布局**: 侧边常驻工具条（目录跳转 + 功能按钮）

## 页面布局建议

- **布局模式**: 单栏纵向滚动 + 右侧悬浮工具条（桌面端）/ 底部Tab工具条（移动端）
- **视觉重心**: 每个模块的核心交互区（二面角滑块模型、长方体拆解、题库等）占据模块主视觉区域，文字说明辅助
- **结果承载区**: 答题即时反馈区（每题下方嵌入对错提示 + 解析展开）；错题本独立面板（抽屉式从右侧滑出）
- **源材料承载区**: 立体几何模型区（SVG/CSS实现，支持交互操作），与文字讲解左右并列（桌面）/ 上下堆叠（移动端）

### 模块1：开篇导学·温故知新

- **页面目标**: 通过互动回顾已有知识，自然引出二面角和面面垂直概念
- **功能点**:
  - **垂直层级拖拽拼图**: 3张可拖拽卡片（线线垂直、线面垂直、面面垂直）+ 「研究顺序」轨道的3个放置槽；拖拽到正确位置锁定并变绿，全部正确解锁下一环节并播放成功动效；错误时卡片弹回并弹出提示"想一想：我们研究空间垂直关系的顺序是什么？"
  - **实景触发二面角**: 4张实景图卡片（水坝、墙面、书本、长方体），点击弹出立体标注层（用SVG叠加棱、两个半平面的透明色块 + 标注文字），再次点击关闭
  - **类比思维弹窗**: 「平面角 vs 二面角」对比表格，默认收起，点击展开；对比维度：定义、图形、表示法、范围
  - **课前小测**: 3道单选题，每题选择后即时判分（正确绿色高亮 + 对勾，错误红色 + 叉号 + 展开解析）；全部完成后显示得分

---

### 模块2：新知1 - 二面角核心概念

- **页面目标**: 通过交互模型让学生直观理解二面角定义、平面角作法与性质
- **功能点**:
  - **二面角定义交互模型**: 底部滑块控制两个半平面张角（0°→180°），SVG/CSS 实时渲染半平面开合动画；「平卧式/直立式」切换按钮切换两种画法；点击「显示记法」按钮依次展示 α-l-β、α-AB-β、P-l-Q 三种记法标注
  - **二面角平面角核心交互**: 分步引导（一作二证三求），点击下一步逐步高亮作图过程；棱上有可拖动顶点，拖动时平面角两边始终垂直于棱，角度数值实时更新但大小不变，直观展示"平面角与顶点位置无关"；0°~180°滑动条旁实时显示二面角类型标签（锐/直/钝/平），颜色随类型变化
  - **教室实景互动题**: 教室墙面地面示意图，点击选择二面角的棱和平面，填空输入角度数，提交后自动批改并给出解析
  - **例题分层交互**: 正方体模型题 - 点击棱自动生成两条垂线并高亮平面角；四棱锥模型题 - 分步解锁证明过程（点击"下一步"显示一步推导）
  - **易错点弹窗**: 2个高频误区卡片，点击翻转显示正确解释（①"任意夹角都是平面角"→ 必须两边都垂直于棱；②"顶点必须取中点"→ 顶点可在棱上任意位置）

---

### 模块3：新知2 - 面面垂直判定定理

- **页面目标**: 通过可拆解模型和分步证明，让学生掌握面面垂直判定定理
- **功能点**:
  - **长方体可拆解模型**: SVG/CSS 绘制长方体，垂直底面的棱可向上拖动拉出，拉出时同步展开竖直平面，动态展示"一个平面过另一个平面的垂线 → 两平面垂直"；拖动到顶后显示判定结论文字
  - **三种语言切换**: 「文字语言 / 符号语言 / 图形语言」三个切换按钮，点击切换显示对应表述
  - **简记口诀展示**: 「线面垂直 ⇒ 面面垂直」大字卡片，点击有强调动效
  - **定理证明分步动画**: 证明过程分4步，播放按钮自动逐步展示（每步高亮对应图形部分 + 推导文字）；支持暂停、上一步、下一步、回放
  - **模型库工具箱**: 5类高频垂直模型（正方体、四棱锥、长方体、三棱锥、直棱柱）缩略图列表，点击在主展示区加载对应3D/SVG模型，可旋转/展开查看垂直关系

---

### 模块4：方法总结·解题模板

- **页面目标**: 系统化梳理解题方法，形成可复用的解题模板
- **功能点**:
  - **二面角求解三步记忆卡**: 横向排列3张卡片（一作/二证/三求），支持左右滑动切换；每张卡片正面为步骤名称，点击翻转显示详细方法说明
  - **面面垂直证明流程图**: 节点式流程图（找垂线 → 证线面垂直 → 得面面垂直），点击节点展开对应细节说明和例题链接
  - **可缩放思维导图**: 中心节点「面面垂直」向外辐射（二面角定义 → 平面角 → 直二面角 → 面面垂直判定），支持点击放大/缩小、拖拽平移

### 模块5：分层随堂练习

- **页面目标**: 通过分层题库即时检验学习效果，错题自动归集
- **功能点**:
  - **基础层练习**: 选择题 + 填空题（二面角记法辨析、判断直二面角、正方体二面角度数填空），每题提交后即时判分，正确绿色高亮，错误红色提示并展开解析
  - **提升层练习**: 交互作图题（在图上点击标出平面角）+ 步骤填空题（证明过程挖空填写），自动批改
  - **拓展层练习**: 解答大题，分步骤展示标准解析与得分点（点击"查看得分点"展开）
  - **错题本自动归集**: 做错的题目自动存入错题口袋，工具条「错题本」入口可查看所有错题列表，点击跳回原题 + 对应知识点模块
  - **得分统计**: 实时显示当前练习得分、正确率、完成进度

---

### 模块6：课堂小结 + 课后任务

- **页面目标**: 游戏化巩固知识点，布置分层课后任务
- **功能点**:
  - **知识点闯关小游戏**: 5道判断题闯关，答对前进一格，答错后退/扣命；通关后解锁「知识卡片」（核心知识点总结长图）；游戏状态机：idle → playing → won/lost
  - **分层课后任务选择**: 3个任务卡片（基础习题 / 折纸实践 / 拓展任务），可勾选选择，每个任务展开显示具体要求
  - **下一节预习入口**: 「下一节：面面垂直的性质」预告卡片 + 预习提示
  - **课堂总结卡片**: 核心知识点回顾清单，可折叠展开

## 技术选型补充

- **立体几何可视化**: SVG + CSS 3D transforms 实现，不引入 three.js（模型简单、交互以2D平面标注为主，CSS/SVG 足够且性能更优）

# UI 设计指南

## 1. 设计推导依据

- **参考意图**: Free Direction —— 附件无法读取，按需求文档语义自主设计教育风交互课件
- **核心情绪 / 应用类型**: 清晰、可信、有探索感的高中数学立体几何交互课件；工具+内容双属性
- **独特记忆点**: 以"半平面开合"为视觉母题，所有章节头图、滑块、进度条都用两个相交平面的蓝色渐变造型，贯穿二面角概念本身

## 2. Art Direction

- **方向名**: 几何教育蓝
- **Design Style**: Swiss Minimalist + Soft Geometry —— 瑞士式网格保证课堂投影可读性，柔和几何面呼应立体几何主题
- **DNA 参数**: 圆角 subtle（rounded-md） / 阴影 subtle（shadow-sm） / 间距 standard（gap-4 / p-6） / 字体方向 无衬线清晰易读 / 装饰手法 半透明几何面叠加 + 细线标注
- **应用类型**: Tool + Content —— 左侧/底部目录导航 + 中央内容区 + 右侧工具条

## 3. Color System

**色彩关系**: 教育蓝主色 + 同色系浅蓝反馈底 + 冷白阅读背景 + 深墨蓝正文，成功/错误色保持低饱和不刺眼
**配色设计理由**: 蓝色传递理性、专注、可信赖，适合数学课堂长时间阅读；低饱和成功/错误色保证即时反馈但不干扰学习节奏
**主色推导**: 从立体几何"空间、理性、清晰"的语义出发，选中等明度高饱和的教育蓝；通过降低饱和度衍生 accent 与 border
**使用比例**: 65% 中性 / 25% 辅助蓝 / 10% primary；primary 只用于主按钮、当前章节、关键交互高亮

| 角色 | CSS 变量 | Tailwind Class | HSL 值 | 设计说明 |
|---|---|---|---|---|
| bg | `--background` | `bg-background` | hsl(210 40% 98%) | 冷白页面背景，轻微蓝调 |
| card | `--card` | `bg-card` | hsl(0 0% 100%) | 纯白卡片承载题目与模型 |
| text | `--foreground` | `text-foreground` | hsl(222 47% 15%) | 深墨蓝标题正文 |
| textMuted | `--muted-foreground` | `text-muted-foreground` | hsl(215 16% 47%) | 说明文字、辅助标注 |
| primary | `--primary` | `bg-primary` / `text-primary` | hsl(221 83% 53%) | 教育蓝，主交互与品牌锚点 |
| primaryForeground | `--primary-foreground` | `text-primary-foreground` | hsl(0 0% 100%) | 主按钮上的白字 |
| accent | `--accent` | `bg-accent` | hsl(214 92% 95%) | 浅蓝 hover/选中底、几何面填充 |
| accentForeground | `--accent-foreground` | `text-accent-foreground` | hsl(221 83% 35%) | accent 上的深蓝文字 |
| border | `--border` | `border-border` | hsl(214 32% 88%) | 浅蓝灰边界，柔和不突兀 |

**语义色提示**: 
- 成功：hsl(142 55% 45%) —— bg hsl(142 60% 95%) / border hsl(142 45% 80%) / text hsl(142 60% 30%)；低饱和绿，与主色同明度区间
- 错误：hsl(0 72% 51%) —— bg hsl(0 70% 96%) / border hsl(0 60% 85%) / text hsl(0 70% 35%)；低饱和红，避免报警感
- 警告：hsl(38 92% 50%) —— bg hsl(48 95% 95%) / border hsl(40 85% 80%) / text hsl(30 80% 35%)；暖橙，用于提示与易错点

## 4. 字体与节奏

- **font-display**: Noto Sans SC —— 中文标题清晰有力，适合课堂投影远距离阅读
- **font-body**: Noto Sans SC —— 正文笔画均匀，长时间阅读不疲劳
- **字号**: H1 text-3xl ~ text-4xl；H2 text-xl ~ text-2xl；body text-base；muted text-sm；数学公式使用 KaTeX / MathJax 等宽渲染
- **圆角**: 中（rounded-md） —— 卡片与按钮柔和不尖锐，符合教育产品亲和力

## 5. 全局布局契约

- **Reference Layout Use**: 按需求结构推导，6 大模块线性 + 目录双模式导航
- **Page / Section Order**: 首屏封面目录 → 模块1导学 → 模块2二面角 → 模块3面面垂直判定 → 模块4方法总结 → 模块5随堂练习 → 模块6小结作业，与需求 1:1 对齐
- **Standard Content Zone**: `max-w-4xl mx-auto` —— 课件内容区控制在舒适阅读宽度，适配手机到投影
- **Shell / Frame Alignment**: 内容区与框架同宽，侧边工具条（桌面端右侧悬浮 / 移动端底部固定）独立于内容流
- **Padding & Rhythm**: `px-4 md:px-6 py-8 md:py-12`，章节间距 12 单位，模块内卡片间距 4-6 单位
- **Full-bleed Zones**: 首屏封面、章节分隔页可全宽渐变背景，内部文字仍受内容区约束
- **Local Narrowing**: 题目卡片、表单输入可收窄到 `max-w-2xl`，居中呈现
- **Overflow Strategy**: 立体几何 SVG 模型、横向步骤条、模型库使用 `overflow-x-auto` 或缩放适配
- **Flexibility Boundary**: 允许移动端调整卡片内边距、工具条位置（右→底）；不允许切换主色、圆角系统或字体

## 6. 视觉与动效

- **装饰**: 半透明蓝色几何面（模拟半平面）+ 细线标注 + 棱线高亮
- **阴影/边界**: 轻 —— 卡片用 subtle 阴影 + 1px border，模型区用边框+浅底
- **动效**: 精致克制 —— 滑块拖动实时更新角度、平面开合平滑过渡、答题反馈轻弹动、章节切换淡入；所有动效 ≤ 300ms

## 7. 组件原则

- 按钮、题目选项、模型操作点必须有 Default / Hover / Active / Focus-visible / Disabled 状态
- Primary 按钮用于"下一节""提交答案""开始学习"等主行动；Outline 按钮用于切换视图、模型调取；Ghost 用于目录项与工具条
- 题目选项选中态用 accent 底 + primary 边框，正确态用成功色边框+浅绿底，错误态用错误色边框+浅红底+解析展开
- SVG 几何模型：棱用深墨蓝实线、半平面用 15% 透明度蓝色填充、垂线用红色虚线高亮、平面角用橙色弧线标注
- 加载与空状态延续几何面装饰，不用默认占位图

## 8. Image Direction

- **Image Role**: 章节头图 + 实景触发题插图 + 封面背景装饰
- **Image Art Direction**: 极简几何摄影风；低饱和度、冷色调、自然光；构图以建筑/空间中的真实垂直关系为主体（水坝剖面、墙角、翻开的书、长方体建筑），画面留出 40% 负空间供文字叠放；光线柔和侧光，突出面与面的交界棱
- **Image Prompt Keywords**: 水坝侧面剖面 混凝土质感 蓝灰调 / 室内墙角 自然光 白墙木地板 / 硬皮精装书翻开立起 侧面视角 柔和天光 / 现代长方体建筑 极简立面 冷色调 / 几何立体构成 半透明蓝色亚克力板 工作室布光
- **Image Avoidance**: 卡通插画风格、人物素材图、高饱和渐变背景、过度美化的校园场景、与立体几何无关的抽象图案

## 9. Anti-patterns

- **Split personality**: 不同模块用不同主色或圆角；全站统一教育蓝与 rounded-md 系统
- **Default SaaS drift**: 回到通用蓝紫渐变和无意义卡片堆叠；用半平面开合母题贯穿所有视觉
- **Invisible interaction**: 只有 hover 没有 focus-visible；所有可点击的模型点、题目选项、工具条按钮都要有键盘焦点环
- **Mono-hue tyranny**: 按钮、tab、图标、边框、链接全用 primary；按 65-25-10 比例，accent 和中性色承担大部分状态
- **Status color drift**: 成功/错误色饱和度过高刺眼；语义色饱和度与 primary 对齐 ±15%，保持课堂低干扰
- **Tiny touch targets**: 移动端题目选项、模型操作点小于 44px；所有触控目标最小 44×44px
